# KPIマネジメント 学習アプリ — グループコーチング × TTP事例共有会 で対話の場をつくる版

中尾隆一郎『最高の結果を出すKPIマネジメント』（フォレスト出版）を素材に、**「自社のKPI管理術」ではなく「グループコーチング × TTP事例共有会で対話の場をつくる道具箱」** として読み直す、ありえる楽考スタイルの学習アプリ。

## 誰のためのアプリか

**グループコーチング × TTP（Through Transfer Practices／事例共有会）で対話による場づくりをやろうとしている企業**のファシリテーター／マネージャー／OD担当者向け。

- KPIを「上から降ってくる目標」ではなく「チーム全員が同じ信号を見つめて対話するための火種」として扱いたい
- 週次／月次の振り返りを「報告会」から「学習の場」に変えたい
- 他チームの成功を「すごいね」で終わらせず、TTPS（徹底的にパクって進化させる）で自チームに移植したい

## この教材の3つの編集軸

### 1. 「自社のKPI管理 → GC × TTP事例共有会 の対話ツール」への翻訳

原著の各章を、「KPIをどう設定するか」ではなく「KPIをどう対話の火種にするか」に読み替える。各テーマに `【個人版】→【GC×TTP事例共有会版】` の1行変換を明記。

- 【個人版】KPIを1つに絞るべき → 【GC×TTP版】チーム全員が同じ信号を見つめて対話する場をつくる
- 【個人版】PDDS／PDCA／PDSの比較学習 → 【GC×TTP版】週次／月次のPDDS振り返りをチーム対話で回す習慣
- 【個人版】TTPで他社事例を学ぶ → 【GC×TTP版】「〇〇さんのやり方をパクって喜ばれた」を共有する事例共有会

### 2. 中心マイルストーンは「🎯 KPIは信号だから1つ」

複数指標を追いかけるのをやめて、**チーム全員が同じ信号を見つめる**ことができれば、振り返り（Study）はそのまま次の計画（Plan）につながる対話になります。「今、私たちのチームは、どの信号を見つめているか？」の合意が、対話による場づくりの起点。

### 3. 「コンセプト → モデル → ストーリー」の3層構造

- **📐 コンセプト（第1層）** — KPIを『測る道具』から『対話の火種』に読み替える
  - KPIの本質定義／KGI・CSF・KPIの三角形／KPIは信号だから1つ
- **🧩 モデル（第2層）** — CSF絞り込みとPDDS高速化で対話の型を編む
  - CSF絞り込み／目標値の運用性チェック／不安の壁とバカの壁／PDDS高速化
- **🎬 ストーリー（第3層）** — TTPと事例共有で組織を学習する場に変える
  - TTP × TTPS／11業種11の事例／KPI化されていない領域／究極のKPI化

## 中核ループ

各テーマは以下の5ステップで学習：

1. **個人→GC×TTPの変換を読む** — 「【個人版】→【GC×TTP事例共有会版】」の1行で、視点を対話の場づくり側に合わせる
2. **抜き書きを読む** — 原著の該当箇所（重要箇所は **太字**）
3. **実務例を書く** — 自分のチームでの実装に置き換えて言語化
4. **自己評価する** — 4段階ルーブリックで現在地を確認
5. **他アプリで深める** — 関連する3〜5つの他アプリのテーマへリンク

## 想定される使い方（企業導入シナリオ）

### シナリオA：既にGCを始めているチーム
「毎週のGCで話題が広がりすぎる」問題を、**信号1つに絞る**運動から解消。テーマ3をチーム全員で読み、そこから週次PDDSを実装（テーマ7）。

### シナリオB：TTP事例共有会を始めたい組織
テーマ8（TTP × TTPS）とテーマ9（11業種事例）を組み合わせ、月次の事例共有会のプロトコルを設計。事例共有会でパクる対象がCSF（テーマ4）に集中していると、対話の解像度が上がる。

### シナリオC：スタッフ部門を巻き込みたい
テーマ10（KPI化されていない領域）を全社研修の起点にする。事業部門とスタッフ部門が『鍵』の設定ロジックを共有すると、組織横断の対話が生まれる。
