## 📋 前提（何を用意すればいいか）

| # | 用意するもの | 必須／推奨 | メモ |
|---|---|---|---|
| 1 | **本のOCRテキスト** (`.md` or `.txt`) | 必須 | Kindle Live Text で章ごとに抜き書きすれば十分 |
| 2 | GitHubアカウント | 必須 | 無料アカウントでOK |
| 3 | Claudian（Claude Code）または類似のAI環境 | 必須 | このガイドはClaudian前提で書いています |

| 5 | 書籍全体のプレイリストURL（YouTube） | 推奨 | 動画ができてからでOK |

**必須は3つだけです**。動画は後から埋められます。

## 🚀 30分ワークフロー（5ステップ）

### Step 1｜本のOCRを配置する（5分）

Kindle Live Textで章ごとに本文を抜き書き → `参考図書/Kindle/<本のタイトル>/<本のタイトル>.md` に保存。

Obsidianを使っていない場合は、任意の場所に `.md` ファイルで置いてOKです。

### Step 2｜Claudianに本を渡す（1分）

Claudianに、以下のトリガーワードで話しかけてください：

```
「<本のタイトル>」の本を教材にアプリ化してください。
素材：<OCRファイルのパス>
```

Claudianが CLAUDE.md セクション 6-9 の標準ワークフローに従って、次のことを自動でやります：

1. OCRから **主張構造の抽出**（1文で言うと何を主張しているか、主要テーマ8〜12個、3層構造）
2. 各テーマの **設計**（title / lo_tag / seed_question / excerpt / prompt / rubric）
3. **観点クイズの生成**（2〜3問 × 4択、ミラー型フィードバック）
4. **`~/Workspace/ariel-apps/<slug>/` フォルダを作成** し、`data.json` と `index.html` を配置

### Step 3｜動画URLを埋める（10-15分）

各テーマの解説動画は、**NotebookLM** で自動生成できます。

1. NotebookLM で新規ノートブック作成
2. ソースに該当章のMarkdownをアップロード
3. カスタムプロンプト（後述のテンプレ）を貼り付けて「動画解説」を生成
4. 生成URLを `https://www.youtube.com/embed/<VIDEO_ID>` 形式に変換
5. `data.json` の各テーマの `video_embed_url` に埋める

**NotebookLMプロンプトのサンプル**（下記の [[達人のように考える版のプロンプト集]] を参考に自作可能）：
- 動画共通の役割・トーン・出力形式を1つに固定
- 章別プロンプトで「聴き手の現在段階」「次段階への具体アクション」を指定
- 8〜12分の動画

**動画がない状態でも公開できます**（後から埋める運用でOK）。

### Step 4｜デプロイ（5分）

```bash
cd ~/Workspace/ariel-apps
git add <slug>/ index.html
git commit -m "add <slug> learning app"
git push
```

GitHub Pagesが自動でビルド → 1〜3分で公開されます。
公開URL：`https://arieru39.github.io/ariel-apps/<slug>/`

### Step 5｜ランディングに追加する（3分）

`~/Workspace/ariel-apps/index.html` に新しいアプリカードを1つ追加：
- `<div class="app">` ブロックを1つ複製
- タグ・タイトル・説明・リンク先を差し替え
- `.app .tag.<slug>` と `.btn.<slug>` のCSSクラスを追加

Claudianに「ランディングにも <slug> のカードを追加してください」と伝えると、自動で対応します。

## 🔧 トラブルシューティング

| 症状 | 対処 |
|---|---|
| **JSON構文エラー** | `python3 -c "import json; json.load(open('data.json'))"` で検証。ASCII の `"` が本文に紛れていたら日本語括弧 `『』` に置換 |
| **GitHub Pagesが `Deployment failed`** | 空コミットで再試行：`git commit --allow-empty -m "retry deploy" && git push` |
| **アプリが表示されない／localStorageの競合** | ブラウザで DevTools → Application → Local Storage をクリア |
| **動画が埋め込まれない** | `video_embed_url` が `https://www.youtube.com/embed/<VIDEO_ID>` 形式か確認（`watch?v=` 形式ではダメ） |
| **各テーマが動かない** | `data.template.json` に対して不足フィールドがないか、`_template/README.md` のチェックリストで確認 |

## 🎁 テンプレート（`_template/`）

`~/Workspace/ariel-apps/_template/` に **正典テンプレート** があります。

| ファイル | 役割 |
|---|---|
| `index.html` | tamakatsu からの正典コピー。**触らない**（書籍非依存の汎用UI） |
| `data.template.json` | 全フィールドを含むスケルトン。`<プレースホルダ>` を埋めるだけで動く |
| `README.md` | 4ステップの詳細ビルドチェックリスト |

**新規アプリを手動で作る場合**：

```bash
cp -r _template <slug>
cd <slug>
# data.json を編集（data.template.json をリネームして中身を埋める）
```

**改善したくなったら**：**tamakatsu で先に試す → `_template/` に反映 → 他アプリへ横展開** の順で。テンプレートが drift すると量産が壊れます。

## 🌱 さらに進むには

### ルーブリック設計を深めたい
参考：[コンサルタントコーチとしての熟達ルーブリック（3領域 × 4段階 × グラレコ）](https://github.com/arieru39/ariel-apps)
- 3領域を「Seeing / Intervening / Being」の形で設計する
- 各段階に「次段階へのSlow Question」を添える
- L4は到達点ではなく毎日再更新される位相

### NotebookLM解説動画を体系化したい
参考：達人のように考える版の全11章プロンプト集
- 共通プレフィックス（役割・トーン・出力形式・禁止事項）を固定
- 章別プロンプトで「聴き手の現在段階 → 次段階への移行」を指定
- 動画マップ表で全体の構造を可視化

### 自分のGCで使うシナリオを設計したい
参考：バイブコーディング・ギルドのサービステキスト『本棚を、アプリ棚に。』
- 3つの想定シナリオ（GC参加個人／夫婦・親子で並走／職場チームで導入）
- 3ヶ月で全11テーマを1巡する運用

## ⚠️ 大事なこと（読み飛ばさないでほしい3点）

### 1. **アプリは "目的" ではなく "対話の入り口"**
学習アプリは、あなたが誰かと「本を通じて対話する場」を持つための道具です。作って終わり、公開して終わりではなく、**誰かと一緒に触りながら「あなたはどう見立てましたか」を交換する** ためのものです。

### 2. **完璧を狙わずv0.1で出す**
Claudianと組めば30分でv0.1が出せます。テーマが足りない／動画がまだ／説明が薄い、は全部後から直せます。**まず出して、対話を回して、改善する** のサイクルの方が早く学べます。

### 3. **ルーブリックは "評価" ではなく "旅の地図"**
4段階は、参加者を格付けするためではなく、**「今どこにいて、次の一歩はどこか」を可視化する** ためのものです。L4が偉い、L1が劣っている、ではなく、**L1〜L4は生涯行き来する位相** です。
