# 👂 本当に聴く — 再創作としての傾聴 学習アプリ

Landmark Forum 系譜（Werner Erhard × Heidegger × Gadamer）の **「本当の聞く」** の概念と、実践プロトコル **『予想と、実際』** を素材にした、ありえる楽考スタイルの学習アプリ。

**公開URL**: <https://arieru39.github.io/ariel-apps/real-listening/>

## この教材の主張（1文）

**「聞くは受信ではなく行為である。聞き手は聞いている時点で "話し手を自分のところに再創作" してしまっている ── だから "抵抗ゼロで再創作する" 練習をすると、本当に変わるのは聞き手自身になる」**

「わかりました」「なるほど、それは〇〇ですよね」という **理解・要約・言い換え** はすべて、自分の地平（先入見）で相手を引き寄せている ── **再創作の失敗** である。本当に聞くとは、自分の予想を先に置き、「3つの言葉だけ」（もっと聞かせて／他には？／それは、あなたにとってどういうことですか？）で相手の地平が立ち上がるのを待ち、自分の地平が変わるのを引き受ける営みである（Gadamer の言う **地平の融合／Horizontverschmelzung**）。

## 11テーマ × 3層構造

- **第1層 · 前提を裏返す** — `#listening_as_action` / `#understanding_as_failure` / `#list_of_resistances`
- **第2層 · プロトコル『予想と、実際』5段** — `#write_prediction` / `#three_words_rule` / `#speaker_decides` / `#open_prediction` / `#measure_your_change`
- **第3層 · 理論と設計** — `#horizon_fusion` / `#asymptote_not_arrival` / `#other_is_constitutive`

層は積み上げではなく、螺旋的に往復する。**前提が裏返る** → プロトコルが動く → **自分が変わった実感** → もう一段深い前提が裏返る、というループ。

## 中核ループ（プロトコル『予想と、実際』の5段）

```
[入口]     🎯 関係性クイズ（4択・8問）で全体を掴む
   ↓
[① 予想]   相手が何を語るか、5〜10行で書き切る（正解探しではない・自分の地平を可視化するため）
   ↓
[② 3語]    インタビュー中は「もっと聞かせて／他には？／それは、あなたにとってどういうことですか？」の3つだけ使う
   ↓
[③ 委ねる] 話題・順序・時間配分・話し終わりの判断 ── すべて相手に委ねる
   ↓
[④ 開く]   最初に書いた予想を相手に見せて話し合う（自分がどう聞こえたかを開示する）
   ↓
[⑤ 測る]   自分の地平のどこがどう動いたか、1週間追跡する
```

## 学習エンジン（Deliberate Practice ／ Fast & Slow ／ Gadamer ／ Landmark）

このアプリの背骨は、姉妹アプリ [たまかつ](../tamakatsu/) と同じ **達人の実践知エンジン** です：

- **判別ゲーム（4択）** — 「関係性クイズ」8問。書くハードルの前に、選ぶだけで11テーマの関係が掴める
- **層構造ガイド** — 第1層（前提を裏返す）から始める順序を明示。第2層のプロトコルは第1層の裏返しの上に立っている
- **Fast / Slow ルーブリック** — L1〜L4を、Kahneman の System 1（Fast）と System 2（Slow）の位相として読み直し。「Slow で意識化 → 反復で Fast に沈殿 → 文脈で選び分け」の熟達曲線
- **統一原理** — 「**Slow で意図的に "抵抗を検出→封鎖" した経験が、Fast の "そのまま聞ける瞬間" に沈殿する**」（Ericsson『超一流になるのは才能か努力か?』最終章／Kahneman『Fast & Slow』／Gladwell『第1感』／Klein × Kahneman 2009 熟達直観／Gadamer『真理と方法』／Landmark Forum）

## 4択関係性クイズ（8問）— 正解探しではなく「あり方を映す鏡」

いきなり書くのが重かったら、まず4択で全体像を掴めます：

1. `listening_as_action` — 「聞く」は受け身か、行為か
2. `understanding_as_failure` — 「わかりました」は成功か失敗か
3. `three_words_rule` — なぜ3語だけに縛るのか
4. `write_prediction` — 予想を先に書く意味
5. `horizon_fusion` — 地平の融合とは何が起きているか
6. `speaker_decides` — 話題・順序を相手に委ねる理由
7. `measure_your_change` — 変化を測るのは相手か自分か
8. `other_is_constitutive` — なぜ他者は「素材」ではなく「構成要素」なのか

## 5層フィードバック（各選択肢に5つの反響が返る）

- 🔍 **視点／スタンス** — その選択に立ったあなたの構え
- ⚡ **Fast/Slow の位相** — 反射／再構築／身体感覚のどこにいるか
- 💭 **位相メモ** — その位相で起きていることを一行で
- 📖 **本テキストの視座** — Landmark／Gadamer／実践プロトコルからの引用
- 🌱 **立ち止まりの問い** — 一段深めるための問い

## たまかつとの接続点

このアプリは [たまかつ](../tamakatsu/) の以下2テーマと強く結ばれています：

- `#dialogue_and_trust` — グループコーチングは「聞かれる」が答えを引き出す場である
- `#co_creation_of_field` — 場は共に作られる（他者は素材ではなく構成要素）

「本当に聴く」の実践は、たまかつの **価値共創コミュニティ** を動かす基礎筋力です。

## ソース

- **Landmark Forum**：「本当の聞く」の定義文（聞くは行為／抵抗ゼロで再創作する）
